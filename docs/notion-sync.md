# Notion sync

## Required Convex env vars

Set these on the Convex deployment (not in `.env.local`). See [environment.md](./environment.md).

- `NOTION_API_KEY`: internal Notion integration token.
- `NOTION_DATA_SOURCE_ID`: target Notion data source/database id.
- `NOTION_WEBHOOK_VERIFICATION_TOKEN`: token received during the Notion webhook verification handshake.
- `NOTION_SYNC_APP_URL`: public app origin used to build `/object/[id]` links.
- `NOTION_SYNC_FALLBACK_USER_EXTERNAL_ID`: Clerk user id for the fallback sync owner.

## User mapping

- Set `public_metadata.notionSyncEnabled = true` in Clerk for users whose objects should sync.
- Set `public_metadata.notionUserId = "<notion-user-id>"` to map a Notion person to an app user.
- The fallback sync owner must also have `notionSyncEnabled = true`.

## Webhook setup

1. Create a Notion webhook subscription pointing to `/notion-webhook`.
2. When Notion sends the initial verification request, copy the logged `verification_token`.
3. Save that value as `NOTION_WEBHOOK_VERIFICATION_TOKEN`.
4. Paste the same token into the Notion subscription verification dialog.
5. Subscribe to `page.created`, `page.properties_updated`, `page.deleted`, and `page.undeleted`.

## Sync audit

Run the `notionSync/discrepancyReport:reportDiscrepancies` action when you want a read-only check that app Objects and Notion pages still line up. It returns a summary plus a typed discrepancy list and logs one line per issue.

Reported discrepancy kinds:

- `ambiguous_match` — heuristic matching found multiple Notion pages for one Object.
- `unmatched_app_object` — sync-enabled Object with no Notion counterpart.
- `unmatched_notion_page` — Notion page with no matched Object.
- `missing_notion_page` — sync record or match points at a page id that is not in the data source snapshot.
- `link_mismatch` — stored `notionPageId` does not match the page resolved by map link or heuristic matching.
- `field_mismatch` — matched pair has one or more differing sync fields (`differingFields`).
- `missing_sync_record` — matched pair has no `objectNotionSync` row.
- `stale_outbound_hash` — sync row `lastOutboundHash` does not match the current Object fields.

When `summary.inSync` is `true`, both sides are aligned for matching, links, fields, and outbound hashes.

## Sync invariants

- App object links win over heuristic matching. If a Notion page contains a `/object/[id]` map link, sync only treats it as a match when that object exists and is eligible.
- Heuristic matching requires the object name plus at least one shared location/category field, and ambiguous matches are skipped rather than guessed.
- `mapLink` and `internalId` are outbound-only. Notion edits to those fields never patch the app object.
- Tag order does not affect the sync hash. This prevents webhooks generated by equivalent multi-select ordering from causing loops.
- Inbound webhooks whose current Notion field hash equals `lastOutboundHash` are treated as echo events and only refresh sync metadata.
- CSV imports enqueue one outbound sync batch after each successful import batch instead of scheduling a Notion action for every row. Imports use `enqueueOutboundObjectSyncBatchLenient`, which logs and skips individual object failures so one bad sync does not abort the rest of the batch. Interactive object edits still schedule `enqueueOutboundObjectSync` per object.
- Notion API calls retry rate limits and transient server errors before surfacing `lastSyncError`.

## Manual verification checklist

- Create an object in the app as a sync-enabled owner and confirm a Notion page is created.
- Update a synced app object as its owner and confirm the Notion page updates.
- Delete a synced app object as its owner and confirm the Notion page moves to trash.
- Create a page in Notion with a mapped user and confirm the app object is created after geocoding.
- Update a synced Notion page and confirm the app object updates.
- Delete a synced Notion page and confirm the app object is removed.
- Run `reportDiscrepancies` after changes and confirm `summary.inSync` is `true`.
